# proxy-unifi

Запустите прокси-клиент без графического интерфейса прямо на шлюзе UniFi Cloud
Gateway и направляйте в него трафик из штатного интерфейса UniFi
(**Policy Engine → Policy Table**).

proxy-unifi включает **два ядра** — [xray-core](https://github.com/XTLS/Xray-core) и
[sing-box](https://github.com/SagerNet/sing-box) — и автоматически выбирает
подходящее для каждой импортируемой ссылки. Оба завершают **один и тот же** туннель
WireGuard, поэтому в интерфейсе UniFi достаточно **одной** записи VPN Client.

Шлюзы UniFi не имеют встроенной поддержки исходящего прокси и не умеют подключаться к серверам по ссылкам прокси-протоколов (`vless://`, `hysteria2://`, ...).
proxy-unifi закрывает этот пробел **без** изменения пакетов UniFi OS: он представляет
прокси для UniFi как штатно поддерживаемый **WireGuard VPN Client**, который контроллер
уже умеет маршрутизировать. Решение работает без графического интерфейса, управляется
только по SSH (без веб-интерфейса) и сохраняется при перезагрузках и обновлениях
прошивки благодаря [unifi-common](https://github.com/unifi-utilities/unifi-common).

## Быстрый старт

Подключитесь к шлюзу по SSH под пользователем `root` и выполните:

```sh
curl -fsSL https://raw.githubusercontent.com/palmbeachpete9/proxy-unifi/main/install.sh | sh
```

После завершения установки запустите `proxy`, чтобы открыть меню управления:

1. **Импорт / замена прокси-ссылки** — вставьте свою ссылку (например, `https://`, `vless://` и т. п.). Поддерживаются как ссылки на отдельные серверы, так и подписки!

2. **Скопируйте показанную конфигурацию WireGuard VPN Client** — создайте файл `.conf` на своём компьютере и загрузите его в:

   `unifi.ui.com → Ваш шлюз → Settings → VPN → VPN Client → Create New → WireGuard`

3. Наконец, с помощью **Policy Engine** создайте любые нужные маршруты трафика, используя штатные возможности Ubiquiti — маршрутизацию по VLAN / устройству / IP / домену / региону.

Созданный файл `.conf` и конфигурация WireGuard в Ubiquiti сохраняются на всё время работы скрипта. Смена прокси-ссылок или протоколов **не** меняет их, что упрощает обслуживание.

## Как это работает

```
  ┌─────────────────────────── UniFi Cloud Gateway ────────────────────────────┐
  │  клиент VLAN ─▶ маршрут Policy Table ─▶ WireGuard VPN Client (UniFi)       │
  │                              │ зашифрованный WireGuard поверх loopback     │
  │                              ▼  udp 127.0.0.1:51821                        │
  │                  xray-core  ИЛИ  sing-box  (вход WireGuard)                │
  │                              │  завершает туннель и маршрутизирует трафик  │
  │                              ▼  прокси-исход (Ваша импортированная ссылка) │
  └──────────────────────────────┼─────────────────────────────────────────────┘
                                 ▼  в обычный WAN
                              Ваш прокси / VPN-сервер ──▶ Интернет
```

Встроенный в шлюз WireGuard VPN Client выполняет настоящее рукопожатие WireGuard с
активным ядром по loopback. Ядро завершает туннель и направляет весь трафик наружу
через прокси-сервер из Вашей ссылки. Удалённый сервер WireGuard не требуется.

## Совместимость

Пакет совместим с UniFi OS 4.x и новее и проверен на следующих устройствах UniFi:
| Модель | Код |
|---|---|
| UniFi Cloud Gateway Ultra | `UCG-Ultra` |
| UniFi Cloud Gateway Max | `UCG-Max` |
| UniFi Cloud Gateway Fiber | `UCG-Fiber` |
| UniFi Dream Router | `UDR` |
| UniFi Dream Router 7 | `UDR7` |
| UniFi Dream Machine | `UDM` |
| UniFi Dream Machine Pro | `UDM-Pro` |
| UniFi Dream Machine SE | `UDM-SE` |
| UniFi Dream Machine Pro Max | `UDM-Pro-Max` |
| UniFi Express | `UX` |
| UniFi Express 7 | `UX7` |
| UniFi Enterprise Fortress Gateway | `EFG` |

> **Примечание:** Некоторые обновления UniFi OS (например, UniFi OS 5.1.12 с рядом исправлений CVE) могут полностью стереть скрипт и его данные из встроенной памяти.
> Сохраните свою прокси-ссылку на такой случай и переустановите proxy-unifi.

Ядро прокси выбирается автоматически по импортированной ссылке:

| Протокол | Ядро |
|---|---|
| VLESS | xray-core |
| VMess | xray-core |
| Trojan | xray-core |
| Shadowsocks — обычный, AEAD + шифры 2022 | xray-core |
| Shadowsocks + obfs-local / simple-obfs | sing-box |
| Shadowsocks + v2ray-plugin | sing-box |
| Hysteria2 | sing-box |
| TUIC | sing-box |

Одновременно работает только одно ядро. Оба используют **одни и те же** ключи и порт
WireGuard, поэтому единственная запись VPN Client в UniFi работает независимо от того,
какое ядро активно.

> **Примечание:** xray слушает порт WireGuard на loopback (`127.0.0.1:51821`);
> sing-box слушает его на всех интерфейсах (у него нет опции выбора адреса
> прослушивания). Это всё равно безопасно — WireGuard отвечает только одному
> настроенному ключу-пиру.

## Использование

Запустите `proxy` для интерактивного меню или используйте прямые команды:

| Команда | Описание |
|---|---|
| `proxy` | Главное меню |
| `proxy status` | Ядро, настроенный сервер и состояние слушателя |
| `proxy ping [...]` | Проверка ссылки — `...` = `get`·`head`·`tcp`·`icmp` (по умолчанию `get`) |
| `proxy start` · `stop` · `restart` | Управление службой |
| `proxy logs [args]` | Журналы службы (передаются в `journalctl`) |
| `proxy help` | Показать справку |
| `proxy update` | Обновляет бинарники ядер (xray + sing-box) и гео-файлы |

Меню охватывает: импорт/замену ссылки, конфигурацию WireGuard для UniFi,
перегенерацию ключей, изменение порта/MTU/DNS, проверку связи и выбор её протокола,
включение/отключение автозапуска, обновление ядер, обновление гео-файлов и удаление.

## Примечания

- **Управление только по SSH:** нет ни веб-интерфейса, ни управляющего порта,
  доступного из локальной сети, — управление выполняется по SSH командой `proxy`.
- **Loopback-эндпойнт:** WireGuard VPN Client в UniFi указывает на `127.0.0.1:51821`.
  Если интерфейс не принимает loopback-эндпойнт, задайте `WG_LISTEN` в
  `/data/proxy-unifi/etc/settings.env` на маршрутизируемый локальный адрес и
  импортируйте ссылку заново.
- **MTU** по умолчанию `1340` (измените через главное меню, если зависают большие
  передачи); **DNS** по умолчанию `8.8.8.8`.

## Сохранение данных

Все данные хранятся в **`/data/proxy-unifi`**, который UniFi OS сохраняет при
перезагрузках и большинстве обновлений прошивки. Хук загрузки
`/data/on_boot.d/15-proxy-unifi.sh` при каждой загрузке заново создаёт команду
`proxy` и службу systemd (с правильным ядром).

## Обновление

Чтобы обновить функциональность клиента (загрузить файлы с GitHub), просто запустите скрипт снова:

```sh
curl -fsSL https://raw.githubusercontent.com/palmbeachpete9/proxy-unifi/main/install.sh | sh
```

Повторный запуск установщика перезаписывает только `bin/` (скрипт + бинарники) и unit systemd. Ваши ключи WireGuard, добавленная прокси-ссылка и настройки клиента (DNS, MTU) сохраняются — WireGuard-клиент в UniFi продолжает работать без каких-либо изменений.

## Удаление

```sh
proxy            # меню → Uninstall
# или:
rm -rf /data/proxy-unifi   # также стирает ключи/конфигурацию
```
Затем удалите WireGuard VPN Client в интерфейсе UniFi.
