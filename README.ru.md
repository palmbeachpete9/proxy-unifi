# proxy-unifi

Запустите прокси-клиент без графического интерфейса прямо на шлюзе UniFi Cloud
Gateway и направляйте в него трафик из штатного интерфейса UniFi
(**Policy Engine → Policy Table**).

proxy-unifi включает **три ядра** — [xray-core](https://github.com/XTLS/Xray-core),
[sing-box](https://github.com/SagerNet/sing-box) и закреплённое userspace-ядро с
[поддержкой AmneziaWG](https://github.com/amnezia-vpn/amnezia-box). Подходящее ядро
выбирается для импортированных ссылок и сохранённых AWG-профилей. Все три завершают
**один и тот же** туннель WireGuard, поэтому в UniFi достаточно **одной** записи VPN Client.

Шлюзы UniFi не имеют встроенной поддержки исходящего прокси и не умеют подключаться к серверам по ссылкам прокси-протоколов (`vless://`, `hysteria2://`, ...).
proxy-unifi закрывает этот пробел **без** изменения пакетов UniFi OS: он представляет
прокси для UniFi как штатно поддерживаемый **WireGuard VPN Client**, который контроллер
уже умеет маршрутизировать. Решение работает без графического интерфейса, управляется
только по SSH (без веб-интерфейса) и сохраняется при перезагрузках и обновлениях
прошивки благодаря [unifi-common](https://github.com/unifi-utilities/unifi-common).

## Быстрый старт

Подключитесь к шлюзу по SSH под пользователем `root` и выполните:

```sh
curl -fsSL https://raw.githubusercontent.com/palmbeachpete9/proxy-unifi/main/install.sh | sh
```

После завершения установки запустите `proxy`, чтобы открыть меню управления:

1. **Импортируйте подключение** — выберите в меню отдельную серверную ссылку
   (`vless://`, `vmess://`, `trojan://`, `ss://`, `hysteria2://` или `tuic://`),
   HTTPS-подписку, JSON-профиль Xray либо профиль AmneziaWG.

2. **Скопируйте показанную конфигурацию WireGuard VPN Client** — создайте файл `.conf` на своём компьютере и загрузите его в:

   `unifi.ui.com → Ваш шлюз → Settings → VPN → VPN Client → Create New → WireGuard`

3. Наконец, с помощью **Policy Engine** создайте любые нужные маршруты трафика, используя штатные возможности Ubiquiti — маршрутизацию по VLAN / устройству / IP / домену / региону.

Созданный файл `.conf` и конфигурация WireGuard в Ubiquiti сохраняются на всё время работы скрипта. Смена прокси-ссылок или протоколов **не** меняет их, что упрощает обслуживание.

## Как это работает

```
  ┌─────────────────────────── UniFi Cloud Gateway ────────────────────────────┐
  │  клиент VLAN ─▶ маршрут Policy Table ─▶ WireGuard VPN Client (UniFi)       │
  │                              │ зашифрованный WireGuard поверх loopback     │
  │                              ▼  udp 127.0.0.1:51821                        │
  │      xray-core  ИЛИ  sing-box  ИЛИ  ядро AmneziaWG (вход WireGuard)        │
  │                              │  завершает туннель и маршрутизирует трафик  │
  │                              ▼  выбранный прокси- или AWG-исход            │
  └──────────────────────────────┼─────────────────────────────────────────────┘
                                 ▼  в обычный WAN
                              Ваш прокси / VPN-сервер ──▶ Интернет
```

Встроенный в шлюз WireGuard VPN Client выполняет настоящее рукопожатие WireGuard с
активным ядром по loopback. Ядро завершает этот внутренний туннель и направляет
трафик через выбранный прокси- или AWG-исход. В режимах прокси-ссылок удалённый
сервер WireGuard не требуется; в режиме AWG ядро подключается к удалённому пиру
AmneziaWG из выбранного профиля.

## Совместимость

Пакет совместим с UniFi OS 4.x и новее и проверен на следующих устройствах UniFi:
| Модель | Код |
|---|---|
| UniFi Cloud Gateway Ultra | `UCG-Ultra` |
| UniFi Cloud Gateway Max | `UCG-Max` |
| UniFi Cloud Gateway Fiber | `UCG-Fiber` |
| UniFi Dream Router | `UDR` |
| UniFi Dream Router 7 | `UDR7` |
| UniFi Dream Machine | `UDM` |
| UniFi Dream Machine Pro | `UDM-Pro` |
| UniFi Dream Machine SE | `UDM-SE` |
| UniFi Dream Machine Pro Max | `UDM-Pro-Max` |
| UniFi Express | `UX` |
| UniFi Express 7 | `UX7` |
| UniFi Enterprise Fortress Gateway | `EFG` |

> **Примечание:** Некоторые обновления UniFi OS (например, UniFi OS 5.1.12 с рядом исправлений CVE) могут полностью стереть скрипт и его данные из встроенной памяти.
> Сохраните свою прокси-ссылку на такой случай и переустановите proxy-unifi.

Ядро прокси выбирается автоматически по импортированной ссылке:

| Протокол | Ядро |
|---|---|
| VLESS | xray-core |
| VMess | xray-core |
| Trojan | xray-core |
| Shadowsocks — обычный / классический AEAD | xray-core |
| Shadowsocks 2022 — стандартные ключи фиксированной длины | sing-box |
| Shadowsocks 2022 AES-128 — 32-байтный ключ совместимости Xray | xray-core |
| Shadowsocks + obfs-local / simple-obfs | sing-box |
| Shadowsocks + v2ray-plugin | sing-box |
| Hysteria2 | sing-box |
| TUIC | sing-box |
| Конфигурация AmneziaWG 1.5 / 2.0 | ядро AmneziaWG |

Одновременно работает только одно ядро. Все три используют **одни и те же** ключи и порт
WireGuard, поэтому единственная запись VPN Client в UniFi работает независимо от того,
какое ядро активно.

> **Примечание:** xray слушает порт WireGuard на loopback (`127.0.0.1:51821`).
> У sing-box и ядра AmneziaWG нет опции выбора адреса прослушивания внутреннего
> WireGuard-эндпойнта, поэтому они слушают на всех
> интерфейсах — но proxy-unifi добавляет правило `iptables`/`ip6tables`, которое
> отбрасывает все не-loopback пакеты к этому порту, делая его недоступным из
> локальной сети и из интернета. Защита в несколько слоёв: порт закрыт файрволом
> до loopback **и** WireGuard отвечает только одному настроенному ключу-пиру.

## Shadowsocks 2022

SS2022 импортируется через существующие функции одиночных ссылок и подписок;
отдельное меню профилей и вторая запись VPN Client в UniFi не нужны. В каталоге
такие узлы помечаются как `ss2022`, а в разделе **Статус** для активного узла
отображается `Shadowsocks 2022`.

Поддерживаемые стандартные методы:

- `2022-blake3-aes-128-gcm` — 16-байтный PSK в base64
- `2022-blake3-aes-256-gcm` — 32-байтный PSK в base64
- `2022-blake3-chacha20-poly1305` — 32-байтный PSK в base64

Однопользовательские ключи поддерживаются для всех трёх методов. Разделённые
двоеточиями цепочки расширяемой идентификации
(`ключ-реле:ключ-сервера:ключ-пользователя`, до 16 ключей) поддерживаются для двух
методов AES-GCM; эта многопользовательская конструкция несовместима с ChaCha20.
Синтаксис и длина ключей проверяются до загрузки ядра и до изменения активного
туннеля. Стандартные ссылки работают через sing-box. Старый специфичный для Xray
32-байтный ключ AES-128 сохранён через xray-core во избежание регрессий; в подписке
такая строка получает предупреждение о режиме совместимости.

Для защиты SS2022 от повторного воспроизведения часы клиента и сервера должны быть
достаточно точно синхронизированы. Мультиплексирование и UDP-over-TCP не включаются
скрыто: сервер провайдера должен поддерживать их явно, иначе корректная ссылка
перестанет работать. Обычная передача TCP и UDP остаётся включённой по умолчанию.

## Подписки и JSON-пулы

Меню подписок принимает HTTPS-подписки с обычным или закодированным в base64
списком ссылок, а также клиентские JSON-профили Xray с балансировщиками.
Имена узлов сохраняются в Unicode, включая кириллицу и эмодзи. Выбранный узел
проходит тот же проверяемый путь импорта, что и отдельная прокси-ссылка.
Форматы Clash YAML, SIP008, sing-box JSON, WireGuard, AnyTLS, Naive и SSH пока
не разбираются.

При установке создаётся случайный постоянный UUID устройства. Он отправляется
источнику подписки в заголовке `x-hwid`, но не передаётся при перенаправлении на
другой домен. User-Agent по умолчанию — `Happ/2.0`; его можно изменить в
меню для совместимости с провайдером. Постоянный таймер systemd обновляет каталог
с заданным интервалом. Если выбранный узел удалён или найден неоднозначно,
активный туннель остаётся без изменений.

JSON провайдера не запускается без проверки. proxy-unifi проверяет адреса,
ссылки маршрутизации, балансировщики, observatory, DNS и лимиты ресурсов;
удаляет входящие слушатели и управляющие интерфейсы провайдера; затем добавляет
собственный вход WireGuard. Исходящие подключения, правила маршрутизации,
алгоритмы балансировки, проверки доступности и fallback-теги провайдера
сохраняются. Профиль, который нельзя безопасно адаптировать, отклоняется до
запуска xray-core.

## Профили AmneziaWG

Второй блок меню управляет отдельными клиентскими конфигурациями AmneziaWG 1.5 и
2.0. Выберите **Создать новый профиль**, задайте отображаемое имя, вставьте
конфигурацию в `nano` и сохраните её клавишами `Ctrl+X`, `Y`, `Enter`. Сохранённые
профили можно просматривать, выбирать и активировать, редактировать,
переименовывать и удалять; `*` отмечает активный профиль. Кириллица, CJK и эмодзи
сохраняются, а для ограниченных SSH-терминалов UniFi применяется только
отображаемая безопасная замена. `nano` нужен только для создания и редактирования
профилей; остальные функции proxy-unifi доступны и без него.

AWG полностью работает в userspace: не создаёт интерфейс ядра, не меняет
маршруты шлюза, не требует `/dev/net/tun` и не заменяет постоянный WireGuard-клиент
UniFi. Поэтому при выборе AWG-профиля ничего менять в UniFi UI не нужно. Ядро AWG
загружается при первой активации; версия закреплена релизом проекта, а архив
проверяется по встроенному в proxy-unifi SHA-256 для соответствующей архитектуры.

Импортируются поля обфускации `Jc`, `Jmin`, `Jmax`, `S1-S4`, `H1-H4` и `I1-I5`,
включая диапазоны заголовков AWG 2.0 и CPS-шаблоны пакетов. Поддерживаются несколько
пиров, IPv4/IPv6, preshared keys и keepalive. Опасные хуки и команды маршрутизации
`wg-quick` отклоняются. Клиентский `ListenPort` сохраняется в исходном профиле, но
не переносится в рабочую конфигурацию, чтобы не открывать лишний UDP-порт. DNS из
профиля показывается справочно; UniFi получает глобальный DNS из раздела
**Настройки WireGuard**.

## Использование

Запустите `proxy` для интерактивного меню или используйте прямые команды:

| Команда | Описание |
|---|---|
| `proxy` | Главное меню |
| `proxy status` | Ядро, настроенный сервер и состояние слушателя |
| `proxy ping [...]` | Проверка ссылки — `...` = `get`·`head`·`tcp`·`icmp` (по умолчанию `get`) |
| `proxy start` · `stop` · `restart` | Управление службой |
| `proxy logs [args]` | Журналы службы (передаются в `journalctl`) |
| `proxy help` | Показать справку |
| `proxy update` | Обновляет xray, установленные ядра sing-box/AWG и встроенные гео-файлы |
| `proxy geo-update` | Обновляет отдельную базу геоданных маршрутизации |

Меню охватывает: отдельные ссылки, подписки, профили AmneziaWG, JSON-профили Xray, конфигурацию WireGuard для UniFi,
перегенерацию ключей, изменение порта/MTU/DNS, проверку связи и выбор её протокола,
включение/отключение автозапуска, обновление ядер, обновление гео-файлов и удаление.

## Примечания

- **Управление только по SSH:** нет ни веб-интерфейса, ни управляющего порта,
  доступного из локальной сети, — управление выполняется по SSH командой `proxy`.
- **Loopback-эндпойнт:** WireGuard VPN Client в UniFi указывает на `127.0.0.1:51821`.
  Пользовательский адрес `WG_LISTEN` поддерживается для ссылок xray-core и
  JSON-пулов. sing-box и ядро AmneziaWG не умеют привязывать этот вход к заданному
  адресу, поэтому для них proxy-unifi требует loopback-настройку и не запускает
  службу, если защитное правило файрвола установить не удалось.
- **MTU** по умолчанию `1340` (измените через главное меню, если зависают большие
  передачи); **DNS** по умолчанию `8.8.8.8`.
- **Разделение привилегий:** прокси-ядра работают от отдельного пользователя
  `proxy-unifi` в усиленной службе systemd. Импорт и обновление проверяются до
  атомарного изменения состояния; неудачный запуск восстанавливает прошлое
  рабочее состояние.
- **Целостность ядер:** xray-core не устанавливается ниже содержащего исправление
  безопасности `v26.7.11`; для этого релиза и ядра AWG используются встроенные
  хэши для каждой архитектуры. Более новые Xray и sing-box должны совпасть с
  digest GitHub, а Xray дополнительно проверяется по его sidecar-контрольной сумме.

## Сохранение данных

Все данные хранятся в **`/data/proxy-unifi`**, который UniFi OS сохраняет при
перезагрузках и большинстве обновлений прошивки. Хук загрузки
`/data/on_boot.d/15-proxy-unifi.sh` при каждой загрузке заново создаёт команду
`proxy` и службу systemd (с правильным ядром).

## Обновление

Чтобы обновить функциональность клиента (загрузить файлы с GitHub), просто запустите скрипт снова:

```sh
curl -fsSL https://raw.githubusercontent.com/palmbeachpete9/proxy-unifi/main/install.sh | sh
```

При удалённой установке все файлы одного запуска привязываются к одному
неизменяемому коммиту GitHub. Архивы ядер и гео-файлы активируются только после
проверки опубликованных SHA-256; при ошибке обновления восстанавливаются прошлые
файлы и служба.

Повторный запуск установщика перезаписывает только `bin/` (скрипт + бинарники) и unit systemd. Ваши ключи WireGuard, добавленная прокси-ссылка, каталог подписки, профили AmneziaWG и настройки клиента (DNS, MTU) сохраняются — WireGuard-клиент в UniFi продолжает работать без каких-либо изменений.

## Удаление

```sh
proxy            # меню → Uninstall
# или:
rm -rf /data/proxy-unifi   # также стирает ключи/конфигурацию
```
Затем удалите WireGuard VPN Client в интерфейсе UniFi.
